- `.DS_Store` が `git status` に出ていた理由を確認した
- `.gitignore` を作成し、`.DS_Store` を無視対象に追加した

- `git commit -m ".DS_Storeをgitignoreに追加"` を実行した
- 最初の `git push` が拒否される理由を確認した
- `git pull --rebase` を実行して、GitHub側の変更を先に取り込んだ
- もう一度 `git push` を実行して、`.gitignore` の変更をGitHubへ反映した

## 💡 今日の作業で学んでほしかったこと
- 変更確認は、毎回ファイルを1つずつ開かなくても、コミット単位で確認できること
- `.gitignore` を使うと、不要ファイルをGit管理から外せること
- Gitの基本の流れは `add → commit → push` であること
- `push` が失敗しても、`pull --rebase` してからやり直せば進めることが多いこと
- `git status` は作業の現在地確認にとても大事なこと
